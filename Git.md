## Git 기본 흐름

:코드 수정 → add → commit → push

## Git의 주요 특징

1. **버전 관리**

- 프로젝트의 변경 사항 기록, _이전 버전으로쉽게 돌아갈 수 있음._

2. **브랜치와 머지**

- 별도의 작업 공간 _(브랜치)_ 을 만들어 실험적 기능을 개발, 완료 후 메인 프로젝트에 통합 _(머지)_ 할 수 있다.

3. **협업 지원**

- _여러 개발자가 동시에 작업_ 해도 충돌을 관리하고, 피어 리뷰를 통해 코드 품질을 검증 할 수 있다.

## Git 기본 명령어

- **git init** : 현재 디렉토리를 새로운 git 저징소로 초기화.
- **git add** : 파일을 스테이징 영역에 추가.
- _git add <파일명>_ : 변경된 파일을 스테이징 영역에 추가.
- **git commit -m** :"커밋 메세지", 스테이징 변경사항을 커밋하여 로컬 저장소에 기록.
- **git log** : 저장소의 모든 커밋 내역 확인.
- **git puhs** : 로컬 커밋을 원격 저장소에 푸쉬, 다른 팀원들과 공유.
- **git pull** : 원격 저장소의 변경 사항을 로컬로 가져옴.
- **git status** : 현재 작업 디렉토리와, 스테이징 영역의 상태 확인.
