# GitFlow란?

：_GitFlow는 Git을 사용할 때 브랜치를 체계적으로 관리하기 위한 전략이다.  
프로젝트를 보다 구조적으로 운영하기 위해 브랜치의 역할을 명확히 나누는 방식이다._

**대규모 프로젝트나 팀 프로젝트에서 자주 사용된다.**

## GitFlow의 주요 브랜치 구조

GitFlow는 다음과 같은 브랜치로 구성된다.

#### main 브랜치

- 실제 배포되는 안정적인 버전이 저장되는 브랜치
- 항상 정상적으로 동작하는 코드만 존재해야 한다.

#### develop 브랜치

- 개발이 진행되는 브랜치
- 새로운 기능이 추가되기 전 단계의 코드가 모인다.

#### feature 브랜치

- 새로운 기능을 개발할 때 사용하는 브랜치
- develop 브랜치에서 생성된다.
- 기능 개발이 완료되면 develop 브랜치로 병합된다.

#### release 브랜치

- develop 브랜치에서 생성된다.
- 배포 준비 및 버그 수정
- 완료 후 main과 develop에 병합된다.

#### hotfix 브랜치

- 배포 후 긴급 수정이 필요할 때 사용하는 브랜치
- main 브랜치에서 생성된다.

## GitFlow의 작업 흐름

1. develop 브랜치에서 feature 브랜치를 생성한다.
2. 기능 개발을 완료한 후 develop 브랜치에 병합한다.
3. 배포 시점이 되면 release 브랜치를 생성한다.
4. 최종 점검 후 main 브랜치에 병합한다.
5. 긴급 오류 발생 시 hotfix 브랜치를 통해 수정한다.

> 이와 같은 구조를 통해 개발과 배포 과정을 명확히 구분할 수 있다.

**GitFlow는 Git의 브랜치 기능을 활용한 체계적인 협업 전략이다.  
프로젝트 규모와 상황에 맞게 적절히 활용하는 것이 중요하다.**
