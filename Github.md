# GitHub란?

:_Git 기반의 분산 버전 관리 시스템을 지원하며 개발자들과 협업, 소스 코드 공유 등을 할 수 있는, 웹 기반 호스팅 플랫폼._

> 실시간 협업, 코드 공유, 프로젝트 관리를 지원.

`혼자 작업하는 경우에도 백업의 역할을 하며, 팀 프로젝트에서는 코드 공유와 협업을 효율적으로 만들어 준다.`

> > GitHub를 사용하면, 협업이 더욱 편리함.  
> > 개발자라면 알아둬야 하는 플랫폼.

## GitHub의 주요 기능

- **Repository 생성 및 관리**
- **Pull Request**: 코드 변경 제안 및 리뷰.
- **Issue 기능**: 버그 및 개선 사항 관리.
- **Fork 기능**: 다른 사람의 프로젝트를 복사하여 수정 가능.
- **Wiki 및 README 작성**: 프로젝트 설명 문서 작성.
- **협업 기능**: 개발자들과, 코드 공유, 리뷰, 논의 등.

## GitHub 팔로우 기능

- 개념 :
  GitHub에서는 다른 사용자를 팔로우할 수 있다.  
  팔로우를 하면 해당 사용자의 공개 활동 내역을 확인할 수 있다.

### Git과 GitHub의 차이점

:Git은 **버전 관리 도구**이고,  
GitHub는 Git **저장소를 온라인**에서 _관리하는 서비스._
