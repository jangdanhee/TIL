# 각 전공 알아보기

## Frontend

> 사용자가 **직접 보고 상호작용하는** 화면(UI)을 만드는 직군

### 하는 일

- 웹/앱의 **화면 레이아웃** 구성 (버튼, 메뉴, 폼 등)
- **사용자 이벤트 처리** (클릭, 스크롤, 입력 등)
- 백엔드 API에서 데이터를 받아와 **화면에 렌더링**
- **반응형 웹** 구현 (PC, 태블릿, 모바일 대응)
- 웹 **성능 최적화** (로딩 속도, 렌더링 최적화)
- **크로스 브라우저** 호환성 처리 (Chrome, Firefox, Safari 등)
- **접근성(Accessibility)** 고려 (장애인, 고령자 사용 환경)

### 주요 기술 스택

| 분류       | 기술                                  |
| ---------- | ------------------------------------- |
| 기본 언어  | HTML, CSS, JavaScript                 |
| 프레임워크 | React, Vue.js, Angular, Next.js       |
| 스타일링   | Tailwind CSS, Sass, Styled-components |
| 상태 관리  | Redux, Zustand, Recoil                |
| 빌드 도구  | Vite, Webpack                         |
| 테스트     | Jest, Cypress                         |

### 결과물 예시

- 쇼핑몰 상품 목록 페이지
- 로그인 / 회원가입 폼
- 대시보드 차트 UI
- 모바일 앱 화면

## Backend

> 사용자에게 보이지 않는 **서버, 데이터베이스, 비즈니스 로직**을 담당하는 직군

### 하는 일

- **서버 개발**: 클라이언트 요청을 받아 처리하는 서버 구축
- **API 설계 및 개발**: 프론트엔드가 데이터를 요청할 수 있는 통로(REST API, GraphQL) 제공
- **데이터베이스 설계**: 데이터를 어떻게 저장하고 관리할지 구조 설계
- **비즈니스 로직 구현**: 결제 처리, 회원 인증, 추천 알고리즘 등 핵심 기능
- **보안 처리**: 인증(Authentication), 인가(Authorization), SQL Injection 방어 등
- **성능 최적화**: 캐싱, 쿼리 최적화, 로드 밸런싱
- **외부 서비스 연동**: 결제 API, SMS, 이메일, OAuth(소셜 로그인) 등

### 주요 기술 스택

| 분류         | 기술                                          |
| ------------ | --------------------------------------------- |
| 언어         | Java, Python, Node.js, Go, Kotlin             |
| 프레임워크   | Spring Boot, Django, FastAPI, Express, NestJS |
| 데이터베이스 | MySQL, PostgreSQL, MongoDB, Redis             |
| API 방식     | REST API, GraphQL, gRPC                       |
| 인증         | JWT, OAuth2.0, Session                        |
| 테스트       | JUnit, pytest                                 |

### 결과물 예시

- 회원가입/로그인 API
- 상품 데이터 조회/저장 서버
- 주문 처리 시스템
- 알림 발송 서버

## Cloud / DevOps

> 서비스가 **안정적으로 운영**되도록 인프라를 구축하고 배포·관리하는 직군

### 하는 일

- **클라우드 인프라 구축**: 서버, 네트워크, 스토리지 환경 설계 (AWS, GCP, Azure)
- **CI/CD 파이프라인 구축**: 코드가 자동으로 테스트되고 배포되는 시스템 구축
- **컨테이너 관리**: Docker로 앱을 패키징하고 Kubernetes로 오케스트레이션
- **모니터링 및 장애 대응**: 서버 상태, 에러율, 트래픽 실시간 감시 및 알림
- **보안 관리**: 방화벽, IAM(권한 관리), 네트워크 보안 정책 설정
- **비용 최적화**: 클라우드 리소스를 효율적으로 사용해 비용 절감
- **백업 및 재해 복구(DR)** 계획 수립 및 운영

### 주요 기술 스택

| 분류                | 기술                                        |
| ------------------- | ------------------------------------------- |
| 클라우드 플랫폼     | AWS, GCP, Azure                             |
| 컨테이너            | Docker, Kubernetes                          |
| IaC (코드형 인프라) | Terraform, Ansible                          |
| CI/CD               | GitHub Actions, Jenkins, ArgoCD             |
| 모니터링            | Prometheus, Grafana, Datadog                |
| 로그 관리           | ELK Stack (Elasticsearch, Logstash, Kibana) |

### 결과물 예시

- 자동 배포 파이프라인 (코드 push → 자동 테스트 → 서버 반영)
- 트래픽 급증 시 자동 서버 확장(Auto Scaling) 환경
- 무중단 배포(Blue-Green, Canary) 시스템
- 실시간 서버 모니터링 대시보드

## UI/UX Design

> 사용자가 **편리하고 아름답게** 서비스를 사용할 수 있도록 경험과 화면을 설계하는 직군

### 하는 일

- **UX 리서치**: 사용자 인터뷰, 설문, 데이터 분석으로 사용자 니즈 파악
- **정보 구조(IA) 설계**: 메뉴, 페이지 흐름, 콘텐츠 배치 구조 설계
- **와이어프레임**: 화면의 뼈대 레이아웃 스케치
- **프로토타입**: 클릭 가능한 시뮬레이션 제작 (Figma 등)
- **UI 디자인**: 색상, 타이포그래피, 아이콘, 컴포넌트 시각 디자인
- **디자인 시스템 구축**: 일관된 컴포넌트 라이브러리 및 가이드라인 제작
- **사용성 테스트**: 실제 사용자가 화면을 얼마나 잘 사용하는지 검증
- **개발 핸드오프**: 디자인 스펙을 프론트엔드 개발자에게 전달

### 주요 기술/도구

| 분류         | 도구                           |
| ------------ | ------------------------------ |
| UI 디자인    | Figma, Sketch, Adobe XD        |
| 프로토타이핑 | Figma, InVision, Principle     |
| 그래픽       | Adobe Photoshop, Illustrator   |
| 리서치       | Maze, Hotjar, Google Analytics |
| 협업         | Zeplin, Notion, Miro           |

### 결과물 예시

- 앱/웹 화면 디자인 시안
- 클릭 가능한 인터랙티브 프로토타입
- 디자인 시스템 (버튼, 색상, 폰트 가이드)
- 사용성 테스트 리포트

## 5. 프론트엔드와 백엔드의 유기적 협업 체계

프론트엔드(FE)와 백엔드(BE)의 협업은 단순히 데이터를 주고받는 것을 넘어, **계약(Contract) 기반의 병렬 워크플로우**를 지향.

### 협업의 중추: RESTful Interface & Interface Definition

백엔드가 설계한 **API 엔드포인트**는 양측의 개발 규약이 된다. 프론트엔드는 이 규약에 근거하여 사용자 경험을 구현하고, 백엔드는 데이터의 무결성과 비즈니스 로직을 보장.

---

### 엔지니어링 워크플로우 (Standard Collaboration Process)

#### 1. 요구사항 분석 및 스키마 설계 (Requirements & Schema Design)

- 기획 및 디자인 시안을 바탕으로 필요한 데이터 항목을 정의.
- **API 명세화:** Swagger나 Postman 등의 도구를 활용하여 URL, 요청 파라미터, 응답 구조(JSON)를 선제적으로 정의. 이는 양 직군 간의 '기술적 약속'

#### 2. 인터페이스 격리를 통한 병렬 개발 (Parallel Development)

- **Backend:** 설계된 명세에 따라 API 서버를 구축하고 데이터베이스(DB) 및 비즈니스 로직을 구현.
- **Frontend:** 실제 서버가 완성되기 전, 약속된 명세를 기반으로 **Mock Server(가짜 서버)**를 구축하여 UI 및 인터랙션 로직을 독립적으로 개발. 이를 통해 상호 의존성으로 인한 병목 현상을 최소화.

#### 3. 통합 및 상호 운용성 테스트 (Integration & Interoperability Test)

- 백엔드의 API가 완성되면 프론트엔드는 Mock 데이터를 실제 API 엔드포인트로 교체.
- **CORS(Cross-Origin Resource Sharing) 설정:** 보안 정책으로 인한 통신 차단 문제를 해결하고, 데이터가 의도한 규격대로 정확히 전달되는지 검토.

#### 4. 상태 코드를 활용한 예외 처리 (Exception Handling)

통신 결과에 따라 표준 HTTP 상태 코드를 활용하여 지능적으로 대응.

- **200 OK:** 데이터 렌더링 및 사용자 시각화.
- **4xx (Client Error):** 잘못된 요청에 대한 피드백(유효성 검사 등) 제공.
- **5xx (Server Error):** 서버 장애 시 사용자에게 안내 메시지 노출 및 재시도 로직 실행.

## 6. Full-Cycle 협업 생태계 (Ecosystem Flow)

전체 프로젝트는 디자인에서 배포까지 다음과 같은 선순환 구조를 갖습니다.

| 단계        | 프로세스         | 핵심 활동                                           | 관여 직군    |
| :---------- | :--------------- | :-------------------------------------------------- | :----------- |
| **Step 01** | **Design & IA**  | 서비스 정보 구조 설계 및 디자인 에셋 생성           | UI/UX Design |
| **Step 02** | **Contracting**  | API 스펙 정의 및 기술 스택 확정                     | FE, BE       |
| **Step 03** | **Production**   | 클라이언트 UI 구현 및 비즈니스 로직 고도화          | FE, BE       |
| **Step 04** | **Provisioning** | 클라우드 인프라 구성 및 배포 파이프라인(CI/CD) 구축 | Cloud/DevOps |
| **Step 05** | **Validation**   | 통합 테스트, QA 및 성능 모니터링                    | 전 직군      |

> **결론적으로,** 현대의 웹 개발은 **'API 명세'**라는 약속을 중심으로 각 전문 영역이 독립적이면서도 긴밀하게 결합(Loosely Coupled, Highly Cohesive)되는 구조를 지향.

## 정리

- **프론트엔드**: 사용자가 보는 것을 만든다
- **백엔드**: 데이터와 로직을 처리한다
- **클라우드**: 서비스가 24시간 안정적으로 돌아가게 한다
- **디자인**: 사용자가 쉽고 즐겁게 쓸 수 있게 설계한다
- **프론트 ↔ 백엔드**: HTTP + REST API + JSON으로 소통한다
